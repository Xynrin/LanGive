# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

LanGive is a cross-platform LAN file transfer tool built with Wails v2 (Go backend + Vue 3 frontend). It uses mDNS for device discovery and HTTP/WebSocket for transfers. Targets desktop (Windows/macOS/Linux) and mobile (Android/iOS, via gomobile bindings of `internal/`).

Module path: `github.com/Xynrin/LanGive` (Go 1.23). Default service port: `5566`.

## Commands

Development and build go through the Wails CLI from the repo root — it drives both Go and the frontend together. Don't run `go build` or `vite build` directly for app builds.

```bash
# One-time tooling
go install github.com/wailsapp/wails/v2/cmd/wails@latest
cd frontend && npm install && cd ..

# Dev (live reload, frontend on Vite dev server)
wails dev

# Build current platform → build/bin/LanGive
wails build

# Cross-compile (CI uses the same form)
wails build -platform windows/amd64
wails build -platform darwin/arm64
wails build -platform linux/amd64

# Frontend-only iteration
cd frontend && npm run dev    # or: npm run build
```

Mobile builds use `gomobile bind` to compile the `internal/` packages into an AAR, then Gradle assembles the APK — see `.github/workflows/release.yml` for the exact invocation.

There is no test suite yet; `go test ./...` currently has nothing to run.

## Architecture

### Wails bind layer (`main.go`, `app.go`)

`LanGiveApp` in `main.go` is the struct exposed to the frontend via `Bind`. Every public method on it (`GetDevices`, `SendFiles`, `SetPrivacyMode`, etc.) is auto-generated as a JS function under `frontend/wailsjs/` at build time and called from Vue components. To add a frontend-callable backend method, add a method to `*LanGiveApp` — do not add a separate router.

`app.go` defines a separate, currently-unused `App` struct with menu/dialog helpers. The active entry point is `LanGiveApp` in `main.go`; `app.go` is legacy from the Wails template and the dialog methods on `LanGiveApp` are stubs (`SelectFolder`/`SelectFiles` return empty values — they need wiring to `runtime.OpenDirectoryDialog` like `app.go` already does).

The frontend embeds via `//go:embed all:frontend/dist` — `wails build` runs `npm run build` first, so dist must exist before the Go binary links.

### Service layer (`internal/`)

Each subpackage owns one concern and is independently constructable so the same code can be reused from the mobile bindings:

- **`config`** — JSON config at platform-appropriate path (`%LOCALAPPDATA%\LanGive`, `~/Library/Application Support/LanGive`, `~/.config/langive`). Generates a stable `DeviceUUID` on first run. `GetScanInterval()` returns a different interval based on `BackgroundMode` — foreground 5s, background 30s, device timeout 15s.
- **`mdns`** — Registers `_langive._tcp.local` and browses for peers every 5s. Identity travels in TXT records (`name`, `platform`, `version`, `uuid`, `session`, `privacy`). `SetDeviceName`/`SetPrivacy`/`SetSession` call `restart()` to re-publish — any TXT-record change requires a full mDNS restart.
- **`transfer`** — Runs a Gin HTTP server on `cfg.Port` with `POST /upload` (receive), `GET /transfers`, `POST /cancel/:id`, and `GET /ws` (progress broadcast). Sending uses `multipart/form-data` POST to `http://<peer>:<port>/upload`. Folders are zipped to a tempfile then sent as one file.
- **`security`** — `Manager` tracks sessions and HMAC-style device tokens (24h validity). Currently constructed and a public session created at startup, but the transfer layer does not yet check tokens — wiring is incomplete.
- **`updater`** — Stub file (currently empty); `LanGiveApp.CheckUpdate` references it but will need implementing.

### Session model

Two visibility modes, both expressed via the mDNS `session` and `privacy` TXT fields:

- **Public session** (`session=public`, `privacy=0`) — discoverable to all non-private peers. `GetPublicDevices()` filters to this set.
- **Privacy mode** (`privacy=1`, `session=<random-uuid>`) — `config.SetPrivacyMode(true)` swaps `SessionID` to a fresh UUID so the device disappears from public discovery. Toggling privacy must be followed by `mdns.SetPrivacy()` so the broadcast restarts (see `LanGiveApp.SetPrivacyMode`).

### Frontend (`frontend/`)

Vue 3 + Vue Router + Pinia, built with Vite. Four routes (`Home`, `Devices`, `Transfers`, `Settings`) under `src/views/`. Calls into Go via the generated `wailsjs/go/main/LanGiveApp` module — regenerated by `wails dev`/`wails build`, not hand-edited (and gitignored per `.gitignore`).

## Conventions

- User-facing strings (menu items, dialogs, comments in Go and Vue) are in Chinese. Match that when extending.
- Code comments are bilingual: function-level Go doc comments are in Chinese (`// 启动服务`), exported signatures in English. Follow the existing pattern when adding code to `internal/`.
- The design doc at `docs/SPEC.md` is the source of truth for protocol details (TXT record schema, port, scan intervals, config path table). Update it when those change.
- Tags `v*` trigger the full release matrix in `.github/workflows/release.yml` (Windows/macOS/Linux × archs, plus Android via gomobile). `-ldflags "-X main.Version=..."` is set from the tag — but note the const `config.Version` is the actual source read at runtime, so bumping a release means updating both.
