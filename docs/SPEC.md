## 一、项目概述

1. **项目名称**：LanGive
2. **项目目标**：
   构建一个跨平台的局域网内设备自动发现、文件传输及资源共享工具，支持主流系统（Windows、macOS、Linux、iOS、Android），实现高效、安全的本地交互。
3. **核心功能**：
   - 自动发现局域网内设备
   - 设备间基于"会话隔离"的安全通信，默认拥有一个公共会话
   - 会话自动续接：程序运行期间持续扫描设备，后台运行时减少扫描频率，降低资源占用
   - 支持自定义设备实例名称（默认使用设备名）
   - 支持下载路径自定义
   - 支持不加入公共会话的隐私模式

## 二、技术栈

- **后端**：Go
- **前端 UI**：wails
- **打包**：GitHub Actions
- **Logo 生成**：oh-my-logo
- **设备发现协议**：mDNS
- **更新核心**：Git + Releases
- **更新逻辑**：仓库版本对比

---

## 三、系统架构设计

### 1. 整体架构图

```text
前端 UI ↔ 后端服务
    ↓
mDNS 设备发现层
    ↓
设备A（Windows/Linux） ↔ 设备B（macOS/iOS/Android）
```

### 2. 模块划分

- **设备发现模块**：基于 mDNS 协议实现设备自动探测与识别
- **传输模块**：基于 TCP/IP 协议实现设备间文件传输
- **存储模块**：处理文件/文件夹的发送与下载，支持断点续传
- **安全模块**：会话隔离（仅在局域网内传输，无公网暴露）
- **传输进度模块**：处理文件/文件夹的发送与下载进度，同时在前端展示
- **更新模块**：通过 Git 版本对比或 Releases 包内更新（不调用 GitHub API）
- **配置模块**：管理实例名称、下载路径等用户设置

## 四、技术选型与实现细节

### 1. 语言与框架

- **Go 后端**：高性能、跨平台，适合网络服务开发
- **wails 前端**：快速构建跨平台 UI，兼容多平台（移动端适配已预留，防止 UI 排版错误）

### 2. 打包与部署

使用 GitHub Actions 自动构建不同平台的可执行文件，支持全平台发布。

### 3. 跨平台支持

| 平台    | 架构/格式                                                  |
| ------- | ---------------------------------------------------------- |
| Windows | amd64 / x86 / arm64                                        |
| macOS   | amd64 / arm64（Apple Silicon）                             |
| Linux   | amd64 / x86 / arm64，打包格式支持 AppImage / RPM / deb     |
| iOS     | arm64                                                      |
| Android | arm64 / 386 / amd64                                        |

### 4. 协议实现（mDNS）

- 使用 mDNS 服务发现协议（兼容 Bonjour/Avahi），实现设备自动注册与发现
- 支持自定义实例名称（如"我的电脑"），避免 IP 冲突

### 5. 更新机制

- **版本对比更新**：后端拉取 Git 仓库版本号，触发更新流程；使用 changelog 记录日志，更新时提示用户更新
- **自动更新流程**：从 Releases 下载最新版本包，自动完成更新
